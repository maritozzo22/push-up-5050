# Plan 04.4-02: Generate Adaptive Icons

**Phase**: 04.4 - Android Adaptive Icon
**Wave**: 2
**Autonomous**: true
**Depends on**: 04.4-01

## Objective

Generate Android Adaptive Icons using flutter_launcher_icons for all densities.

## Context

After configuring pubspec.yaml with the adaptive icon settings, we need to run flutter_launcher_icons to generate:
- ic_launcher.xml files for mipmap-anydpi-v26
- PNG files for all legacy densities (hdpi, xhdpi, xxhdpi, xxxhdpi)

## Tasks

### Task 1: Run flutter_launcher_icons
Execute the icon generation command:
```bash
cd push_up_5050
dart run flutter_launcher_icons
```

This will:
1. Read pubspec.yaml configuration
2. Generate adaptive icon XML files in mipmap-anydpi-v26/
3. Generate PNG files for all densities

### Task 2: Verify generated files
Check that the following files were created in `android/app/src/main/res/`:

**mipmap-anydpi-v26/** (adaptive icons):
- ic_launcher.xml
- ic_launcher_round.xml

**Density directories** (legacy icons for older Android versions):
- mipmap-hdpi/ic_launcher.png (72x72)
- mipmap-hdpi/ic_launcher_round.png
- mipmap-mdpi/ic_launcher.png (48x48)
- mipmap-mdpi/ic_launcher_round.png
- mipmap-xhdpi/ic_launcher.png (96x96)
- mipmap-xhdpi/ic_launcher_round.png
- mipmap-xxhdpi/ic_launcher.png (144x144)
- mipmap-xxhdpi/ic_launcher_round.png
- mipmap-xxxhdpi/ic_launcher.png (192x192)
- mipmap-xxxhdpi/ic_launcher_round.png

### Task 3: Verify ic_launcher.xml content
Check that mipmap-anydpi-v26/ic_launcher.xml contains:
- Background color: #FF6B00
- Foreground drawable reference
