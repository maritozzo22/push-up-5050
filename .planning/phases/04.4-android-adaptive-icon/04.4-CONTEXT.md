# Phase 04.4: Android Adaptive Icon - Context

**Gathered:** 2026-01-30
**Status:** Ready for planning

<domain>
## Phase Boundary

Implement Android Adaptive Icon system so the app icon displays correctly across all Android launcher shapes (round, square, teardrop, squircle) while maintaining brand identity.

**Includes:**
- Adaptive icon configuration (AndroidManifest, XML resources)
- Foreground layer (logo) and background layer design
- Generating icons for all screen densities
- Legacy fallback for pre-Android 8.0 devices

**Excludes:**
- iOS app icon changes (separate platform)
- Notification icon changes (covered in Phase 04.5)
- Splash screen or launcher screen changes

</domain>

<decisions>
## Implementation Decisions

### Visual design
- **Foreground layer:** Use the original colored PUSHUP 5050 logo (no white/color conversion)
- **Background layer:** Orange gradient (subtle gradient using orange tones for depth)
- **Logo source:** Use existing project logo assets

### Icon generation
- **Densities:** Generate for all 5 standard Android densities (mdpi, hdpi, xhdpi, xxhdpi, xxxhdpi)

### Testing approach
- **Primary method:** Use Android emulator launcher mock feature for testing various shapes
- **Rationale:** Physical device testing requires multiple devices with different launchers; emulator provides efficient validation

### Claude's Discretion
- **Safe zone margins:** Standard Android safe zone guidelines (one-third margin from edges)
- **Foreground decorative elements:** Whether to include subtle container around logo
- **Icon generation tool:** Android Studio Image Asset Studio vs automated scripts
- **Source logo format:** PNG vs SVG based on existing logo availability
- **Project structure:** Follow Android Studio default for mipmap organization
- **Legacy icon approach:** How to handle Android < 8.0 (API 25 and below)
- **Backward compatibility:** Whether to maintain full support for older Android versions
- **Legacy icon visual treatment:** How legacy icon relates to adaptive icon
- **Resource qualifiers:** Whether to use separate v26+/v25- configurations
- **Launcher testing prioritization:** Which launcher mocks to prioritize
- **Automated testing:** Whether to include automated checks for adaptive icon XML
- **Test documentation:** Whether to formally document testing results

</decisions>

<specifics>
## Specific Ideas

- Brand orange #FF6B00 should be used as base for gradient background
- Original colored logo should be prominently visible on all launcher shapes
- Emulator launcher mocks are preferred over physical device testing for efficiency

</specifics>
