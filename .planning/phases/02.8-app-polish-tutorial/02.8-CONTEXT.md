## Implementation Decisions

### Onboarding Structure

**Page count: 3 pages**
- Page 1: Welcome + Philosophy (app illustration/hero image, title, tagline)
- Page 2: How it Works + slight app tour (icons + text list for features, tour Statistics and Workout/Series screens)
- Page 3: Goal configuration

**Navigation pattern:**
- Swipe between pages + left/right arrow buttons
- Dot indicator at bottom showing current page position
- NO skip button - onboarding is required (full gate)

**Localization:**
- Must support BOTH Italian and English versions
- User language detection based on device locale

**Welcome page layout:**
- Illustration/hero image at top
- App title and tagline below
- "Continue" button to proceed

**How It Works content:**
- Icons + text list format for each feature:
  - Progressive series counting
  - Streak tracking
  - Points system
  - Achievement system

**App tour scope:**
- Statistics page - show progress tracking
- Workout/Series page - show how to start and count reps
- Home page mentioned briefly
- Achievements and Settings not shown in tour

### Goal Configuration UX

**Daily goal input:**
- +/- buttons that increment by 1
- Long-press accelerates: faster and faster (like existing workout rep selector)
- No upper limit (unlimited range)
- Default: 50 push-ups

**Monthly goal:**
- Auto-calculated from daily (daily Ã— 30)
- Always editable by tapping the number
- Manual override persists

**Progress preview:**
- Show dynamic time estimate: "At this pace, you'll reach 5050 in X days/months/years"
- Updates as user adjusts daily goal

**Post-onboarding goal changes:**
- Settings screen includes option to change daily/monthly goals anytime
- Same +/- button pattern for consistency

### First Launch Behavior

**Detection method:**
- SharedPreferences flag: `onboarding_completed`
- Check on app launch in main.dart

**Gate behavior:**
- Full gate - user MUST complete onboarding before accessing any app features
- No bypass, no dismissal option

**Re-show option:**
- Settings screen: "Riavvia tutorial" / "Restart tutorial" button
- Resets the `onboarding_completed` flag
- User goes through full flow again

**Persistence:**
- `onboarding_completed` flag set after completing Page 3 (goals)
- Daily and monthly goals saved to SharedPreferences
- Goals loaded by UserStatsProvider on app start

### UI Overflow Fixes

**Affected screens (user-reported):**
- Statistics screen
- Workout Setup (Series Selection) screen
- Other screens: no overflow issues reported, but verify all

**Fix approach:**
- Claude's discretion - investigate root cause of overflow on these specific screens
- Not all pages have this issue - fix only where needed
- Options: SingleChildScrollView, layout resizing, padding adjustments

**Minimum screen support:**
- 4.7 inches and up

**Testing sequence:**
- Implement all overflow fixes first
- Then test all screens together on emulator (not per-screen testing)
